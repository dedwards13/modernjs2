What is a Pattern?

  • A reusable solution that can be applied to occurring problems in software design (JavaScript Applications)
  • Can also be thought of as programming templates
  • Situations vary significantly

Module

Used for overall structure. Allows us to break our code up into modules or we can have private or public functions & variables

// Basic Module structure
(function() {
    // Declare private vars & functions
    return {
       // Declare public vars & functions
    }
})();

Revealing Module Pattern

Singleton

The Singleton pattern is kind of a variation of the module pattern where we can only create one instance of an object,

A Singleton object is an immediate anonymous function, and it can only return one instance of an object at a time. Now, an example of why you may want to use a singleton is maybe you only want, for instance, one user object created at a time, maybe a logged in user. It would prevent you from having two users from being created at once. Now, oftentimes, singletons are frowned upon because they give you a global point of access rather than embracing encapsulation.

Factory

The factory pattern is used to create many, many objects.

Also called the factory method in JavaScript, which is a way of creating an interface for creating objects. But we can let subclasses define which classes to instantiate.And factory methods are often used in applications to manage and maintain and manipulate collectionsof objects that are different, but at the same time have many common characteristics.

Observer

The observer pattern allows us to basically subscribe and unsbuscribe to events.

It can be used to notify the dome of certain elements to be updated and actually angular JS relies heavily on this pattern through event management within the scope.
